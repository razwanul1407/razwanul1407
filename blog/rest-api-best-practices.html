Best Practices for REST API Integration in Mobile Apps

Md Razwanul Hasan

April 18, 2025  ·  11 min read

A leaky API layer is the silent killer of mobile apps. It causes crashes, blank screens, data corruption, and frustrated users. After integrating dozens of APIs across 21+ apps, here's the pattern I use that eliminates nearly all of these problems.

1. Use a Typed HTTP Client, Never Raw http

In Flutter, I use Dio. In React Native, Axios. Both support interceptors, timeout configuration, and request cancellation out of the box. Never use the raw http package or fetch directly in your repository — you lose too much control.

// Dio setup with interceptors (Flutter)
final dio = Dio(BaseOptions(
  baseUrl: 'https://api.yourapp.com/v1/',
  connectTimeout: const Duration(seconds: 15),
  receiveTimeout: const Duration(seconds: 30),
));

// Auth interceptor
dio.interceptors.add(InterceptorsWrapper(
  onRequest: (options, handler) async {
    final token = await SecureStorage.getToken();
    if (token != null) {
      options.headers['Authorization'] = 'Bearer $token';
    }
    return handler.next(options);
  },
  onError: (error, handler) async {
    if (error.response?.statusCode == 401) {
      // Refresh token logic here
      final refreshed = await refreshToken();
      if (refreshed) return handler.resolve(await _retry(error.requestOptions));
    }
    return handler.next(error);
  },
));

2. Always Model Your Responses

Never use raw Map<String, dynamic> throughout your app. Create model classes with fromJson() factories. This catches breaking API changes at the model parsing layer, not in your UI.

class UserModel {
  final int id;
  final String name;
  final String email;
  final String? avatar;

  const UserModel({
    required this.id,
    required this.name,
    required this.email,
    this.avatar,
  });

  factory UserModel.fromJson(Map<String, dynamic> json) {
    return UserModel(
      id: json['id'] as int,
      name: json['name'] as String,
      email: json['email'] as String,
      avatar: json['avatar_url'] as String?,
    );
  }
}

3. Wrap Every API Call in a Result Type

API calls can fail in many ways: no internet, server error, timeout, parsing error. Use a Result/Either type to force callers to handle failures explicitly.

typedef ApiResult<T> = Either<Failure, T>;

abstract class Failure {
  final String message;
  const Failure(this.message);
}

class NetworkFailure extends Failure {
  const NetworkFailure() : super('No internet connection. Check your network.');
}

class ServerFailure extends Failure {
  final int statusCode;
  const ServerFailure(this.statusCode)
      : super('Server error ($statusCode). Please try again.');
}

class ParseFailure extends Failure {
  const ParseFailure() : super('Unexpected data received from server.');
}

4. Implement Proper Error Handling in the Repository

Future<ApiResult<User>> getUser(int userId) async {
  try {
    final response = await dio.get('/users/$userId');
    final model = UserModel.fromJson(response.data);
    return Right(model.toEntity());
  } on DioException catch (e) {
    if (e.type == DioExceptionType.connectionError) {
      return const Left(NetworkFailure());
    }
    return Left(ServerFailure(e.response?.statusCode ?? 500));
  } catch (e) {
    return const Left(ParseFailure());
  }
}

5. Cache Aggressively, Sync Wisely

Mobile users have intermittent connectivity. Never assume the network is available. My strategy:

  • Show cached data immediately while fetching fresh data in background
  • Use Hive or SQLite for structured local caching in Flutter
  • Set TTL (Time To Live) on cached entries — not everything should be cached forever
  • For list screens: cache the list. For detail screens: cache by ID.
Future<ApiResult<List<Product>>> getProducts() async {
  // 1. Return cached data immediately
  final cached = await localCache.getProducts();
  if (cached != null && !cached.isExpired) {
    return Right(cached.data);
  }

  // 2. Fetch fresh data
  final result = await remoteDataSource.getProducts();
  result.fold(
    (failure) => null, // Use stale cache if network fails
    (products) => localCache.saveProducts(products),
  );

  return result.isRight() ? result : Right(cached?.data ?? []);
}

6. Handle Token Refresh Transparently

JWT tokens expire. Handle this in your Dio/Axios interceptor, not in your screens. The user should never see an authentication error — it should be refreshed silently behind the scenes.

7. Pagination: Use Cursor-Based, Not Offset

For long lists, use cursor-based pagination (pass the last item's ID or timestamp) rather than offset-based (?page=2&per_page=20). Offset pagination has race conditions when items are inserted — users can see duplicates or miss items.

8. Implement Request Deduplication

If a user taps a button quickly, don't fire duplicate API requests. Cancel the in-flight request before firing a new one, or debounce the trigger.

A robust API layer is invisible to the user — they just experience a fast, reliable app. That's exactly what we're building for.
