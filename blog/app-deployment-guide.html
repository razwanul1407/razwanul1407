From Code to Store: App Deployment Guide

Md Razwanul Hasan

August 10, 2025  ·  12 min read

I've submitted 21+ apps to both the Google Play Store and Apple App Store Connect, with a 95%+ first-submission approval rate. Here's everything I know about getting your app live without the painful back-and-forth with reviewers.

Before You Build: Account Setup

Google Play Console

  • One-time $25 developer fee
  • Business account vs Personal: use Business if you're shipping commercial apps
  • Set up Google Play App Signing immediately — don't manage your own keystore if you can avoid it
  • Enable 2-factor authentication

Apple App Store Connect

  • $99/year Apple Developer Program fee
  • You need a Mac (or CI/CD service) to build iOS apps
  • Set up your App Store Connect account and Xcode certificates early — Apple's certificate system can be confusing
  • Create a Bundle ID in the Apple Developer portal before starting your project

Flutter: Building Release APK/IPA

# Android (App Bundle — required for Play Store)
flutter build appbundle --release

# iOS (Archive via Xcode or command line)
flutter build ipa --release

Always use App Bundle (.aab) for Play Store — not APK. Google Play will generate optimized APKs per device automatically, reducing download sizes by 20–40%.

Signing Your App

Android Keystore

Generate once and never lose this file. If you lose your keystore, you cannot update your app on Play Store — you'll need to create a new app listing.

keytool -genkey -v -keystore my-release-key.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias my-key-alias

Store the keystore in a secure location outside your project. Add it to your .gitignore. Store credentials in environment variables or a key.properties file (also gitignored).

Play Store Submission Checklist

  • ✅ App Bundle signed with release keystore
  • ✅ Target API level meets Google's current requirement (API 34+ as of 2025)
  • ✅ App icon: 512×512 PNG, no alpha transparency on the corners
  • ✅ Feature graphic: 1024×500 PNG
  • ✅ Screenshots: at least 2 per device type (phone, tablet if applicable)
  • ✅ Privacy policy URL (required for any app requesting permissions)
  • ✅ Content rating questionnaire completed honestly
  • ✅ Data safety form filled out accurately
  • ✅ Short and full description written (no keyword stuffing)

App Store Submission Checklist

  • ✅ Archive built and validated in Xcode with no errors/warnings
  • ✅ App icon: 1024×1024 PNG, no alpha channel, no rounded corners (Apple adds them)
  • ✅ Screenshots for all required device sizes (iPhone 6.7", 6.5", iPad Pro if needed)
  • ✅ App description: max 4000 chars, no markdown, no placeholder text
  • ✅ Keywords: 100 chars max, comma-separated
  • ✅ Privacy policy URL
  • ✅ Support URL
  • ✅ Age rating set correctly
  • ✅ In-app purchases configured before submission if applicable
  • ✅ TestFlight build passes internal testing before App Store submission

Top Reasons Apps Get Rejected

Play Store

  • Privacy policy missing (most common)
  • Permissions requested but not used or not explained
  • Targeting old API levels
  • Misleading screenshots or description

App Store

  • Broken functionality during review (test on a real device first)
  • Placeholder content left in ("Lorem ipsum", test data)
  • No privacy policy for apps that collect data
  • Login required with no demo credentials provided in review notes
  • App duplicates existing App Store functionality without added value

Tip: Add Review Notes

For App Store, always fill in the "Notes for App Review" field. If your app requires login, provide a test account. If it uses special hardware, explain how it works. Apple reviewers use this — it dramatically reduces rejections from misunderstandings.

Phased Rollout

Both stores support phased rollout. Use it for major updates — release to 10% of users first, monitor crash rates, then expand. I've caught post-release bugs this way before they hit 100% of my users.

The most expensive bug is one that reaches all your users simultaneously. Phased rollout is free insurance.
